元素常见布局

发表于 2018-03-05 | 分类于 CSS

水平居中

行内元素

  • text-align: center

这种方法可以让 inline/inline-block/inline-table/inline/flex 等类型的元素实现居中。

块级元素

  • margin: 0 auto (宽度确定)
  • text-align + inline
  • flex + justify-content
  • table + margin

table是块级元素，但是宽度默认为内容实际撑开的宽度。

  • float + relative
  • absolute + transform

left: 50%是指子元素左边框相对于父元素左边框右移父元素宽度的50%，translateX(-50%)把元素沿着横轴移动自身宽度的50%，左侧为开始点即0点，-50%即向左移动自身宽度的50%.

垂直居中

行内元素

  • line-height + height (单行文本)
  • table-cell + vertical-align (多行文本)
  • flex + align-items

块级元素

  • absolute + 负margin (高度确定)
  • absolute + transform (高度未知)
  • flex + align-items

水平垂直居中

宽高固定

  • absolute + 负margin

未知宽高

  • flex + justify-content + align-items
  • absolute + transform

缺陷：当计算结果含有小数时，会让整个元素看起来是模糊的。解决方案：父元素设置 transform-style：preserve-3d

  • table-cell + inline-block
  • table-cell + absolute

grid 布局 justify-items/align-items：center

两列布局

两个div并列排列

  • 浮动
  • inline-block
  • table
  • position + margin

左侧定宽，右侧自适应

  • float + margin
  • float + overflow
  • absolute + margin
  • flex
  • table-cell

右侧定宽，左侧自适应

与前者方式一样，注意：元素的排列顺序，右侧放在前面，左侧放在后面。

等高布局

  • flex
  • float + 负margin-bottom
  • table-cell

左右定宽，中间自适应

  • 圣杯布局

区别： 圣杯布局和双飞翼布局基本上是一致的，都是两边固定宽度，中间自适应的三栏布局，其中，中间栏放到文档流前面，保证先行渲染。解决方案大体相同，都是三栏全部float:left浮动，区别在于解决中间栏div的内容不被遮挡上，圣杯布局是中间栏在添加相对定位，并配合left和right属性，效果上表现为三栏是单独分开的（如果可以看到空隙的话），而双飞翼布局是在中间栏的div中嵌套一个div，内容写在嵌套的div里，然后对嵌套的div设置margin-left和margin-right，效果上表现为左右两栏在中间栏的上面，中间栏还是100%宽度，只不过中间栏的内容通过margin的值显示在中间。

  • 双飞翼布局

双飞翼布局是在middle的div里又插入了一个div，通过调整内部div的margin值，实现中间栏自适应，内容写在内部div中。

  • absolute + margin
  • flex
  • float + margin

注意元素排列顺序，按照：left、right、middle

  • table-cell
